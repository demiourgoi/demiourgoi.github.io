io.github.demiourgoi.linoleum.evaluator

SpanStreamEvaluator

Companion object SpanStreamEvaluator

class SpanStreamEvaluator extends Function[SpanInfoStream, VerifiedTraceStream] with Serializable

For each span in a trace we emit a SpanStart and SpanEnd event, and order all the events using the span start time for SpanStart events and the end time for SpanEnd events. We then discretize the ordered sequence using tumbling windows of tickPeriod duration. The first letter always starts with SpanStart(rootSpan) and events before that are discarded as errors.

Note a limitation is that if the clocks in the different hosts that emit the spans are too skewed then we could lose the happens-before relation between parent and child spans when we order by event timestamp, that is inherited from the span timestamps. This is a known limitation to be tackled on https://github.com/juanrh/Linoleum/issues/4

If a span gets into the event more than once (two SpanInfo have the same spanId) then we only process one of them. One of the replicas is chosen arbitrarily, so this assumes the trace instrumentation libraries only use the same span id for identical spans.
